MIT Policy Guidelines

  • The MIT letterforms should be displayed on the site’s top page, preferably every page, and should link to the MIT home page, http://web.mit.edu. Download Web-ready letterforms for your choice of background color, or obtain them from the Publishing Services Bureau (PSB).
  • Third-party advertising or commercial promotion is not allowed on MIT web sites. They can create conflicts of interest and potentially jeopardize MIT's tax-exempt status. Sponsorship should comply with MIT's sponsorship guidelines.
  • Departments and Organizations selling goods and services over the web should follow MIT e-commerce guidelines.
  • If you use a copyright statement, the copyright is to "MIT" rather than a department.
  • All websites on the MIT domain should comply with MIT's legal and policy guidelines.

Target Browsers & Platforms

  • Netscape Navigator 4.x and above (Windows, MacOS, Unix)
  • Microsoft Internet Explorer 5.x and above (Windows, MacOS)
  • Lynx (UNIX)

Accessibility Guidelines

Pages should comply with MIT accessibility guidelines. These are based on the World Wide Web Consortium's (W3C) Web Accessibility Initiative (WAI).

Technical and Coding Considerations

  • Frames should not be used.
  • Cascading Style Sheets may be used when they are compatible across browsers.
  • Pages should be functional without Java, Javascript, CSS, Flash or browser specific tags. These features can be used to enhance functionality, but must not be required, and the page should degrade gracefully when they are not present. For example, you can use Javascript to create navigation rollovers as long as a user can access all information on the site with Javascript turned off.
  • HTML should be compliant with version 4.0 standards, or validated by a clean load into Dreamweaver. The world wide web consortium (W3C) provides an online HTML validator.
  • Each file should include a doctype statement; e.g., <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
  • The head of each document should contain a meaningful title, followed by keyword and description meta tags for better search engine indexing.
  • HTML filenames should be lowercase, with no spaces, for Unix compatibility
  • Tables should be coded with proportional widths (i.e., stretchable to accomodate a variety of monitor resolutions)
  • If a link opens a new browser window, subsequent requests for a second window should target that same window (rather than open up a new window with each request).
  • The pages should be designed for a screen resolution of 800 pixels by 600 pixels. To achieve this, all primary elements (navigation, main content, etc.) must be no wider than 760 pixels, and the page should not lose data when printed with default settings.

Graphics and Color

  • Pages should load fairly quickly. Test loading on a 56KB modem.
  • The minimum color depth is 8 bit (256 colors); please test the screens at 256-color depth.
  • Animated images should be kept to a minimum in occurrence and length of time. It is our experience that the MIT community prefers to see moving images upon request rather than imposed upon them.

Fonts

  • Font face should degrade gracefully (code with alternative choices).
  • As a general guideline, body text should be default size. Relative size –1 can be used for menu items, quotes, and copyright information.
  • When using CSS to specify font size, avoid using PX, since it prevents users who need large type from resizing the text in their browser. It is fine to use PT (greater than 10 PT), percentages, or small/medium/large.

Printing

Pages should be printable both in terms of page contrast and page width.

Features of web.mit.edu

Services provided:

  • Web publishing space and nightly backup of the data.
  • Forms (generic script available for feedback forms and surveys).
  • Search engine: Ultraseek can be configured to search just within a specific directory; for example, your locker.
  • Web reporting: we provide an "invisible counter" to track web hits.
  • MIT has a streaming video service.
  • Server-side includes.
  • Copyright and Commercial activity.
  • Campus map.
  • Events calendar.

Services not provided:

MIT does not allow third-party CGI scripts on its server.
